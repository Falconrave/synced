InheritanceEmbeddedSwfLoader flash.events.EventDispatcher

Allows you to load an embeded SWF stored as a ByteArray then access any stored classes within the SWF. An Event.COMPLETE will be dispatched upon the successful completion of a call to load. IOErrorEvent.IO_ERROR will be dispatched if there's a problem reading the ByteArray. Embed your swf like this (take note of the mimeType!): 
 [Embed(source="foo.swf", mimeType="application/octet-stream")]
 private static const FOO_RESOURCE :Class;
Then: 
 var loader :EmbeddedSwfLoader = new EmbeddedSwfLoader();
 loader.addEventListener(Event.COMPLETE, handleComplete);
 loader.addEventListener(IOErrorEvent.IO_ERROR, handleFailure);
 loader.load(FOO_RESOURCE);

public function EmbeddedSwfLoader(useSubDomain:Boolean = false)

Create an EmbeddedSwfLoader, good for loading one SWF from a ByteArray.

Parameters
useSubDomain:Boolean (default = false) — if true, load the SWF and all its classes and symbols into a child ApplicationDomain. This means that they would be not normally reachable by the rest of your code after they're loaded, you would only be able to access the contents using this EmbeddedSwfLoader. This also means that those classes or symbols won't interfere with others loaded with the same name, and the classes can be garbage collected when no longer in use.

public function load(byteArrayOrClass:Object):void

Load the SWF from a ByteArray. A COMPLETE event will be dispatched on successful completion of the load. If any errors occur, a IO_ERROR event will be dispatched.

Parameters
byteArrayOrClass:Object — a flash.utils.ByteArray or a Class object that becomes one.

Throws
— The parameter must be a ByteArray, or a Class that becomes one. If AS had overloading, this wouldn't be necessary, we could catch these problems at compile time.
